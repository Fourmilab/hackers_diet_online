Field Definitions

Date
Date of the entry in the legacy U.S. format “M/D/YY”. For example, July 20th, 1999 is written as “7/20/99”. When importing CSV files in Excel formats, the ISO 8601 format of “YYYY-MM-DD” and the European legacy format of “DD.MM.YYYY” are accepted as well. Two digit years are assumed to be in the 1900s if 89 or greater (I started The Hacker's Diet in 1989) and in the 2000s otherwise.
Weekday
Day of the week in English. This field is exported for compatibility with the Excel spreadsheet design and ignored on import.
Weight
Daily weight, as a decimal number. If no weight was recorded for the day, this field must contain a non-blank, non-numeric value; it may be used as a comment. When exporting databases in this format, days with no weight entry and no comment are indicated by a value of “---”.
Trend
Current trend value as a decimal number. This field is exported for compatibility with the Excel spreadsheet layout and ignored on import.
Variance
Current variance (weight−trend) as a signed decimal number. This field is exported for compatibility with the Excel spreadsheet layout and ignored on import.
Last Trend
Last trend value for a day with a weight specified. This is a decimal number field which is used in trend computation in the Excel spreadsheet and kept in a hidden column. It is exported purely for compatibility and ignored on import.
Rung
Rung in the exercise program completed for this day. This is a whole number between 1 and 48, blank if no exercise was done on that day.
Flag
This field sets the “Flag” checkbox in the Monthly Log, which you can use to record any significant daily yes or no item (for example, did you work out at the gym that day, or have pizza for lunch?). If the field is blank or “0” the flag is not set; a value of “1” sets it.
